- 调查按以下顺序逐类检查，每类有信号则定位到具体行，无信号则跳过，不得因某一类信号突出就省略其余类别：

category（必须使用 C1-C7）：

| 值 | 含义 | 典型信号 | 判断边界 |
|----|------|---------|----------|
| C1 | Host Scheduling | DataLoader worker=0、CPU 单线程、GPU 等待数据 | — |
| C2 | Kernel Launch Overhead | Python 循环触发大量小 CUDA op | — |
| C3 | Synchronization | `.item()`、`cudaDeviceSynchronize()`、blocking copy | 须有 D2H/sync profile 信号或明确 blocking call，不凭 kernel 名称推断；`.cpu()`/`.numpy()`/`.tolist()`/`.item()` 在训练热路径内触发时归 C3（主要危害是强制 host/device 同步），仅当 profile D2H bytes 显著且无明显 sync 信号时才考虑 C4 |
| C4 | Memory Copy | `.to()`、`.cuda()`、H2D/D2H、缺少 pinned memory | 多行独立拷贝各自输出 finding；`.cpu()`/`.numpy()` 同步语义优先判 C3，见上 |
| C5 | Compute Inefficiency | 低效算子、重复计算、不必要的 clone/cat | — |
| C6 | Communication | NCCL all_reduce/all_gather、通信未重叠 | 须确认源码有 `init_process_group` 且 profile StringIds 含 `nccl` |
| C7 | Framework / Python Pipeline | tokenizer、collate、Python 序列化、checkpoint 热路径 | 须定位到具体函数行，不要泛指 DataLoader；`json.dumps`/`json.loads`/Python 序列化在热路径内归 C7，不是 C3 |
